# PRD – Personal Tech Blog

## 1. 개요

- 제품 이름: 개인 기술 블로그
- 버전: v1.0
- 작성일: 2026-06-12
- 작성자: [Your Name]

### 1.1 목적

개발/AI/생산성 관련 글과 튜토리얼을 정리하여, 나 자신과 비슷한 배경의 독자들이 검색/회고하기 쉬운 공간을 제공한다.

### 1.2 배경

- 여러 사이드 프로젝트, AI 실험, 개발 팁이 여기저기 흩어져 있어 재사용/공유가 어렵다.
- GitHub / Notion / 블로그 포스트를 한 곳에 모으는 허브가 필요하다.

### 1.3 목표

- 1주 안에 최소 기능(MVP) 블로그를 배포한다.
- 첫 달 안에 최소 5개의 글을 게시한다.
- 검색엔진에서 내 이름 + 키워드(예: “이름 + Claude + GitHub Actions”)로 쉽게 찾을 수 있도록 한다.

## 3. 범위 (Scope)

  - 마크다운 기반 글 작성
  - 코드블록 하이라이트
  - 태그(예: AI, Dev, Productivity)
- 기본 SEO:
  - 페이지별 title/description
  - Open Graph / Twitter 카드 메타 태그

## 4. 핵심 기능 요구사항

### 4.1 홈 / 리스트 페이지

**요구사항**

1. 최신 글 목록이 발행일 역순으로 표시되어야 한다.
2. 각 글 카드는 다음 정보를 포함한다.
   - 제목
   - 발행일
   - 태그
   - 짧은 요약(1~2문장)
3. 상단에는 간단한 자기소개 섹션과 SNS/GitHub 링크가 노출되어야 한다.

**수용 기준**

- 새 글을 1개 추가하고 빌드/배포했을 때, 홈에서 가장 상단에 나타난다.
- 카드 클릭 시 해당 글 상세 페이지로 이동한다.

---

### 4.2 포스트 상세 페이지

**요구사항**

1. 마크다운 기반으로 글을 작성할 수 있어야 한다.
2. 코드 블록은 언어에 따라 syntax highlight가 적용되어야 한다.
3. 이미지 삽입 및 캡션(선택)을 지원한다.
4. 상단에 제목 / 발행일 / 태그가 노출된다.
5. 글 하단에 “이전 글 / 다음 글” 내비게이션을 제공한다 (선택).

**수용 기준**

- 마크다운 파일에 코드블록과 이미지가 있을 때, 브라우저에서 보기 좋은 형태로 렌더링된다.
- 태그를 클릭하면 해당 태그의 글 목록 페이지로 이동(선택 구현).

---

### 4.3 태그/카테고리

**요구사항**

1. 각 글에 하나 이상의 태그를 지정할 수 있어야 한다.
2. 태그 페이지에서 해당 태그를 가진 글들의 목록을 볼 수 있어야 한다.

**수용 기준**

- 특정 태그를 가진 글을 2개 이상 추가했을 때, 태그 페이지에서 모두 조회된다.

---

### 4.4 테마 / 디자인

**요구사항**

1. 기본 다크 테마 적용 (DESIGN.md 규칙을 따른다).
2. 본문 가독성에 최적화된 레이아웃 (max-width ~720px 수준).
3. 모바일/태블릿/데스크톱에서 모두 깨지지 않고 읽기 편해야 한다.

**수용 기준**

- 주요 브라우저(Chrome, Edge, Safari)와 모바일에서 레이아웃이 깨지지 않는다.
- 본문 폰트, heading, 코드블록 스타일이 DESIGN.md에 정의한 토큰과 일치한다.
